The RemoteFirst pattern (e.g., RemoteFirstPythonBridge) means each adapter first attempts to call the remote service. On network error, timeout, or non-2xx response, it throws an exception that is caught by the agent's error handler, which then skips that skill execution and continues. The Python bridge and FAISS index are separate Node.js processes or containerized microservices — they are not co-located in the same process as the Fastify server.
